# 03 — 学习路线图 (Scientific Learning Roadmap)

> 本学习路线采用 **「闯关式」** 设计——每个阶段有明确的目标、必做的操作和通关检验标准。  
> 请 **严格按顺序** 推进，不要跳级。

---

## Stage 1: 走马观花（基础认知）

> **目标**：建立全局心智模型，知道"谁管什么"，能够口述出一次技能释放的完整调用链。  
> **预计阅读量**：5 个核心文件，约 1200 行代码。

### 必读清单（按阅读顺序）

| 顺序 | 文件 | 为什么最重要 |
|------|------|-------------|
| ① | `Structs/SceneVariants.cs` | **全局"门面"**。30 秒内你就能看完它——全是静态方法转发。但从这里你能一次性认识系统中所有子系统的 **公开 API**：`CreateBullet`、`CreateAoE`、`CreateDamage`、`CreateTimeline`、`CreateCharacter`……它是整个项目的"目录"。 |
| ② | `GameManager.cs` | **入口点 + 工厂**。`Start()` 是游戏的启动流程，`CreateCharacter()` / `CreateBullet()` / `CreateAoE()` 是所有实体的诞生地。读完你就知道游戏一开始做了什么。 |
| ③ | `Character/ChaState.cs` | **角色的"大脑"**。这是全项目最大、最核心的类（~590 行）。属性计算、Buff 管理、技能释放、移动/动画指令分发全在这里。它是所有系统的"交汇点"。 |
| ④ | `Structs/GameTimeline.cs` | **Timeline 数据结构定义**。这个文件只有 ~160 行，但定义了 `TimelineObj` / `TimelineModel` / `TimelineNode` / `TimelineGoTo` 四个结构。理解它们等于理解了"技能效果是如何被描述的"。 |
| ⑤ | `DamageManager.cs` | **伤害流水线**。只有 ~100 行代码，但浓缩了整个战斗结算的核心逻辑。读完你就能回答："一次伤害是怎么从子弹命中走到 HP 减少的？" |

### 通关检验

## Stage 2: 抽丝剥茧（核心机制深度解读）

> **目标**：深入理解本项目最精彩、设计含金量最高的 3 个模块。  
> **心态**：不要急着看懂每一行，先理解"它试图解决什么问题"，再看"它是怎么做的"。

### 模块 A：数据驱动的回调委托系统

**涉及文件**：`Structs/Battle/Bullet.cs` (BulletModel) + `GameData/DesignerScripts/Bullet.cs` + `GameData/DesignerTables/Bullet.cs`

**这个模块精彩在哪**：  
`BulletModel` 是一个 `struct`，它的字段中有 3 个 **委托（delegate）** —— `onCreate`、`onHit`、`onRemoved`。策划通过字符串查字典（`DesignerScripts.Bullet.onHitFunc["CommonBulletHit"]`）拿到函数引用，填进 Model。框架代码（`BulletManager`）在正确的时机调用这些委托。

**阅读指导**：

1. 先打开 `Structs/Battle/Bullet.cs`，找到 `BulletModel` 的定义，标注出所有 delegate 字段。
2. 注意构造函数：字符串参数如何映射为函数引用的：
   ```csharp
   this.onHit = onHit == "" ? null : DesignerScripts.Bullet.onHitFunc[onHit];
   ```
3. 打开 `DesignerTables/Bullet.cs` 看策划怎么填表——注意 `new BulletModel(...)` 的参数排列。
4. 打开 `DesignerScripts/Bullet.cs` 看 `CommonBulletHit()` 的具体实现——它从 `onHitParams` 数组中取参数。
5. 最后打开 `BulletManager.cs`，看框架是在哪个时机调用 `bs.model.onHit(bullet, character)` 的。

**思考题**：  
> 如果你要新增一种"穿透子弹"（穿过第一个敌人继续飞，命中 3 个目标才消失），你需要修改哪些文件？需要改框架代码吗？  
> （答案：不需要。只需在 `DesignerTables/Bullet.cs` 新增条目，设置 `hitTimes = 3`。）

### 模块 B：Buff 系统的 8 种回调时机

**涉及文件**：`Structs/Character/Buff.cs` (BuffModel) + `Character/ChaState.cs` (AddBuff/Buff管理) + `DamageManager.cs` + `GameData/DesignerScripts/Buff.cs`

**这个模块精彩在哪**：  
一个 `BuffModel` 居然有 **8 个回调时机**，覆盖了角色战斗行为的方方面面：

| 回调 | 触发时机 | 典型用途 |
|------|---------|---------|
| `onOccur` | Buff 被添加/层数变化时 | 播放特效、初始化状态 |
| `onTick` | 每 `tickTime` 秒 | DoT 伤害、周期性治疗 |
| `onRemoved` | Buff 被移除前 | 清理特效、传送角色 |
| `onCast` | 携带者释放任何技能时 | 拦截技能、替换 Timeline（见 AutoReload） |
| `onHit` | 携带者的攻击命中敌人时 | 吸血、追加伤害 |
| `onBeHurt` | 携带者被攻击时 | 减伤、反弹、只受 1 点伤害 |
| `onKill` | 携带者击杀目标时 | 击杀回血、连杀计数 |
| `onBeKilled` | 携带者被击杀时 | 死亡爆炸（炸药桶就是这么实现的） |

**阅读指导**：

1. 在 `Buff.cs` 找到 `BuffModel` 的 8 个委托字段定义。
2. 在 `DamageManager.DealWithDamage()` 中，用注释标注 `onHit` / `onBeHurt` / `onKill` / `onBeKilled` 的调用位置。
3. 在 `ChaState.FixedUpdate()` 中找到 `onTick` / `onRemoved` 的调用位置。
4. 在 `ChaState.CastSkill()` 中找到 `onCast` 的调用位置——注意它能 **返回一个不同的 Timeline**（这是 AutoReload 的关键）。
5. 重点阅读 `DesignerScripts/Buff.cs` 中的 `ReloadAmmo()`（onCast 拦截换弹）和 `OnlyTakeOneDirectDamage()`（beHurt 只受 1 伤害）。

**思考题**：  
> `onCast` 回调返回 `null` 会发生什么？（答案：`CastSkill()` 中 `timeline == null` → 技能释放被取消。）  
> 这等于 Buff 可以「禁止玩家释放特定技能」——用数据而非硬编码实现了"沉默"效果。

### 模块 C：Timeline 帧事件调度器

**涉及文件**：`Structs/GameTimeline.cs` + `TimelineManager.cs` + `GameData/DesignerScripts/Timeline.cs` + `GameData/DesignerTables/Timeline.cs`

**这个模块精彩在哪**：  
它用一个极其简单的结构（时间点 + 函数指针 + 参数数组），表达了游戏中所有技能效果的"编排"。这不是 Unity 的 Timeline，而是一种 **命令式的帧事件队列**，类似 Flash 时代的"关键帧"概念。

**阅读指导**：

1. 先读 `GameTimeline.cs` 理解数据结构：
   - `TimelineModel`：策划填的原始数据（nodes + duration + chargeGoBack）
   - `TimelineNode`：单个事件（time + 函数引用 + params）
   - `TimelineObj`：运行时实例（model + caster + timeElapsed + timeScale）
   - `TimelineGoTo`：蓄力循环跳转点
2. 读 `TimelineManager.FixedUpdate()`——核心只有 ~40 行，但它驱动了整个技能系统。
3. 在 `DesignerTables/Timeline.cs` 中选择 **`skill_roll`**（翻滚技能）逐行解读——它是最复杂的 Timeline：
   ```
   t=0.00  播放翻滚动画（根据方向）
   t=0.00  挂载火焰特效
   t=0.01  锁定移动/转向/技能
   t=0.10  设置无敌 0.7 秒
   t=0.20  强制位移 2 米 / 0.5 秒
   t=0.80  移除火焰特效 + 播放冲击波
   t=0.80  恢复操控
   t=0.90  Timeline 结束
   ```
4. 读 `DesignerScripts/Timeline.cs` 中的 `CasterForceMove()` 理解"强制位移"是如何通过 `MovePreorder` 实现的。

## Stage 4: 破坏与重构（融会贯通）

> **目标**：通过亲手添加新功能，验证你是否真正理解了架构。  
> **难度**：★★★☆☆（中等）  
> **预计修改文件数**：3~4 个，全部在 `GameData/` 目录内，**不需要修改任何框架代码**。

### 任务：实现一个新技能 —— "冰冻光波"

**技能设计**：
> 角色面前释放一个 AoE 区域，持续 3 秒。区域内的敌人每 1 秒受到一次伤害（攻击力的 0.5 倍），并且移动速度降低 50%。

**实现步骤提示**（不是答案，是线索）：

1. **在 `DesignerTables/Buff.cs` 新增一个 "Frozen" BuffModel**：
   - `propMod[0]`（加法层）设置 `moveSpeed = -50`（减速 50 点）
   - `tickTime = 1.0f`
   - `onTick` → 你可以新写一个，或者复用 `BarrelDurationLose` 的思路
   - `maxStack = 1`

2. **在 `DesignerTables/Aoe.cs` 新增一个 "FreezeZone" AoeModel**：
   - `prefab`：你可以随便用一个现有的特效或留空
   - `onChaEnter` → `"DoDamageToEnterCha"` 或者新写一个专门加 Buff 的函数
   - `tickTime = 1.0f`，`onTick` → 对范围内角色造成伤害 + 刷新 Frozen Buff

3. **在 `DesignerTables/Timeline.cs` 新增一个 "skill_freezeWave" TimelineModel**：
   - 参考 `skill_spaceMonkeyBall` 的结构
   - `t=0.00` → `SetCasterControlState` 锁定技能
   - `t=0.10` → `CreateAoE`（你的 FreezeZone）
   - `t=0.50` → `SetCasterControlState` 恢复

4. **在 `DesignerTables/Skill.cs` 新增一条 "freezeWave" SkillModel**：
   - `cost = ChaResource.Null`（无消耗）
   - `effectTimeline = "skill_freezeWave"`

5. **在 `GameManager.Start()` 中给主角 `LearnSkill`** + 在 `PlayerController` 绑定一个按键。

### 通关检验

- [ ] 新技能能正常释放，AoE 出现在角色面前。
- [ ] 进入 AoE 的敌人被添加了 "Frozen" Buff，移动速度明显降低。
- [ ] AoE 持续 3 秒后自动消失，敌人的 Buff 随后到期移除，移速恢复。
- [ ] **你没有修改 `BulletManager.cs`、`AoeManager.cs`、`DamageManager.cs`、`ChaState.cs` 中的任何一行代码** —— 这就是数据驱动架构的威力。
